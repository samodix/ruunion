# Instructions agents — RU Union

Toujours répondre en français.

## Règles générales

- Toujours faire un audit avant modification.
- Toujours protéger les secrets.
- Ne jamais committer `.env`, credentials, dumps SQL, backups, uploads complets, `wp-config.php`, `node_modules` ou `.next`.
- Toujours faire un backup avant SFTP.
- Toujours tester `npm run lint` et `npm run build` quand des fichiers applicatifs Next.js ou config concernée changent.
- Toujours produire un rapport final factuel.
- Ne jamais force-push.

## Projet RU Union

- WordPress/EasyWP est la cible finale.
- Next.js reste prototype/source UI.
- Aucun Elementor MCP n’est requis pour le nouveau workflow.
- Aucun paiement réel WooCommerce sans confirmation explicite.
- Ne jamais modifier `wp-config.php`.
- Ne jamais utiliser les credentials DB dans `wp-config.php`.
